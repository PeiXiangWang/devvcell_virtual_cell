# SwarmLineage-OT: A Developmental Branch-Window Order-Parameter Framework

SwarmLineage-OT converts native OT-inferred developmental pseudo-lineage maps into executable finite-agent virtual-cell dynamics. The retained manuscript story is a developmental time-series branch-window order-parameter framework centered on transient condensation-before-divergence.

The strongest evidence remains internal native moscot and E1 MouseGastrulationData. A final independent embryoid-body and spatial/time-series sprint did not upgrade the external atlas above weak support: GSE154572 lacks curated lineage labels and STDS0000074/GSE123187 is not branch-window-ready in the inspected cell-level form. Therefore the manuscript should state that broader cross-system generalization remains unresolved.

Clone fate prediction, topological-neighbour specificity, swarm-required causality, birth/death, memory, CCI and diffusion as an independent discovery are excluded from the main claim.

We further audited whether branch windows become more stable in GRN/regulon space. Native pySCENIC/GRNBoost2 and CellOracle-style tools were not available in the current environment, but the project reuses PathwayFinder's local OmniPath CollecTRI/DoRothEA TF-target priors, with TF-target co-expression used only as a fallback when prior targets are absent. This recovered candidate developmental TF programs but still did not produce clean internal/E1 negative-control support for a generalized regulatory branch-window mechanism. The GRN layer is therefore retained as a regulatory audit and candidate-generation module, not as evidence that a causal regulatory mechanism has been established.
